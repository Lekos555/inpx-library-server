# Пошаговая инструкция: как запустить нагрузочный тест

## Что нам понадобится

1. **Второй компьютер** (ноутбук, ПК, Raspberry Pi) в той же сети, что и NAS
2. **Node.js** — программа, которая умеет запускать JavaScript
3. **Файл `load-test.cjs`** из папки `bench/` этого проекта

> ⚠️ **Важно:** тест запускается с ДРУГОГО компьютера, а не с самого NAS. Если запустить на NAS — он будет грузить сам себя и покажет неправильные цифры.

---

## Шаг 1. Узнай IP-адрес своего NAS

### На компьютере с Windows:

1. Открой **Командную строку** (нажми `Win + R`, напиши `cmd`, нажми Enter)
2. Напиши:
   ```
   ping synology
   ```
   или
   ```
   ping 192.168.1.100
   ```
   (замени `192.168.1.100` на реальный IP своего NAS)

3. Если видишь ответы — NAS доступен. Запиши IP-адрес.

### Или через веб-интерфейс Synology:

1. Зайди в DSM (веб-интерфейс NAS)
2. **Панель управления → Сеть → Сетевой интерфейс**
3. Там будет написан IP-адрес, например `192.168.1.100`

---

## Шаг 2. Проверь, что сайт открывается

На втором компьютере открой браузер и перейди по адресу:

```
http://192.168.1.100:3000
```

(замени IP на свой)

Если открылся сайт с книгами — отлично, запомни этот адрес.

---

## Шаг 3. Установи Node.js

### Windows:

1. Иди на сайт: https://nodejs.org/
2. Жми большую зелёную кнопку **"LTS"** (долгосрочная поддержка)
3. Скачай установщик `.msi`
4. Запусти, нажимай **Next → Next → Next → Install → Finish**
5. **Перезагрузи компьютер**

### macOS:

1. Открой **Terminal** (Командная строка)
2. Если установлен Homebrew:
   ```bash
   brew install node
   ```
3. Или скачай с https://nodejs.org/ и установи как обычную программу

### Linux (Ubuntu/Debian):

```bash
sudo apt update
sudo apt install nodejs npm
```

---

## Шаг 4. Проверь, что Node.js работает

Открой командную строку (Windows: `Win + R`, `cmd`, Enter) и напиши:

```bash
node --version
```

Должно вывести что-то вроде:
```
v20.20.2
```

Если пишет "команда не найдена" — Node.js не установился, повтори шаг 3.

---

## Шаг 5. Скопируй файл теста

1. Открой папку проекта `inpx-library-server`
2. Найди папку `bench`
3. Внутри — файл `load-test.cjs`
4. Скопируй этот файл куда угодно на втором компьютере (например, на Рабочий стол)

> 💡 **Совет:** файл `load-test.cjs` — это единственный файл, который нужен для теста. Можно просто перекинуть его через Telegram, Dropbox или флешку.

---

## Шаг 6. Запусти тест

### Windows:

1. Нажми `Win + R`, напиши `cmd`, нажми Enter
2. Перейди в папку, где лежит файл. Если на Рабочем столе:
   ```
   cd Desktop
   ```
3. Запусти:
   ```
   node load-test.cjs http://192.168.1.100:3000
   ```
   (замени `192.168.1.100` на IP своего NAS)

### macOS / Linux:

1. Открой Terminal
2. Перейди в папку:
   ```bash
   cd Desktop
   ```
3. Запусти:
   ```bash
   node load-test.cjs http://192.168.1.100:3000
   ```

---

## Шаг 7. Жди результат

Тест работает примерно **2–5 минут** (зависит от параметров).

В окне командной строки ты увидишь:
```
inpx-library-server load test
Target: http://192.168.1.100:3000
Phases: 1, 5, 10, 20, 30, 50 users
Duration per phase: 15 s

[1/3] Проверка связи...
Сервер отвечает.

[2/3] Сбор ID книг из каталога...
Найдено 24 книг для теста.

[3/3] Нагрузочное тестирование...
```

Потом пойдут строчки с цифрами. Не закрывай окно, пока не увидишь:
```
  ИТОГОВЫЙ ОТЧЁТ
```

---

## Шаг 8. Прочитай результат

В конце выведется таблица вида:

```
  Пользователей │ RPS     │ Ошибок │ p90 latency │ Статус
  ──────────────┼─────────┼────────┼─────────────┼────────────────────
  1             │ 12.5    │ 0      │ 120 ms      │ OK
  5             │ 45.2    │ 0      │ 340 ms      │ OK
  10            │ 78.1    │ 0      │ 890 ms      │ НАГРУЖЕН
  20            │ 65.3    │ 12     │ 5200 ms     │ ДЕГРАДАЦИЯ
```

И в самом конце — **рекомендация**:
```
  РЕКОМЕНДАЦИЯ: сервер стабильно держит ~10 одновременных
  пользователей при комфортной задержке (< 3 с).
```

### Что значит

- **OK** — сайт летает, всё отлично
- **НАГРУЖЕН** — работает, но с задержкой
- **ДЕГРАДАЦИЯ** — начались ошибки, сайт тормозит

Рекомендуемый максимум — последняя фаза без ошибок и с задержкой меньше 3 секунд.

---
## Что тестирует

Скрипт имитирует реальных пользователей со случайным распределением:

- **Главная страница** — 15%
- **Каталог / поиск** — 20%
- **Страница /library/recent** — 15%
- **Страница книги** — 20%
- **Обложка (thumb)** — 15%
- **Скачивание книги** — 10%

## Примеры для разных сценариев

### Быстрый тест (2 минуты)
```
node load-test.cjs http://192.168.1.100:3000 --phases=1,5,10 --duration=10
```

### Только лёгкая нагрузка
```
node load-test.cjs http://192.168.1.100:3000 --phases=1,3,5,7 --duration=20
```

### Тяжёлый тест с полным скачиванием (нагружает диск)
```
node load-test.cjs http://192.168.1.100:3000 --limit-download-bytes=0
```

---

## Если что-то пошло не так

### "Ошибка соединения"
- Проверь, что NAS включен и сайт открывается в браузере
- Убедись, что IP-адрес правильный
- Проверь, что на NAS запущен сервер (контейнер Docker работает)

### "команда node не найдена"
- Node.js не установился или компьютер не перезагружен
- Переустанови Node.js (шаг 3)

### "Cannot find module"
- Файл `load-test.cjs` не в той папке, откуда запускаешь команду
- Убедись, что перешёл в правильную папку командой `cd`

### Тест завис
- Нажми `Ctrl + C` в командной строке — это прервет выполнение
- Попробуй уменьшить количество пользователей: `--phases=1,3,5`
