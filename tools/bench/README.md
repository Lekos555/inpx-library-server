# Нагрузочное тестирование

Скрипт `load-test.cjs` — standalone тестер, не требует `npm install`.

## Запуск

```bash
node bench/load-test.cjs http://192.168.1.100:3000
```

## Что тестируется

Скрипт имитирует реальных пользователей со случайным распределением:

- **Главная страница** — 15%
- **Каталог / поиск** — 20%
- **Страница /library/recent** — 15%
- **Страница книги** — 20%
- **Обложка (thumb)** — 15%
- **Скачивание книги** — 10%

## Параметры

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--phases=1,5,10,20,30,50` | Последовательность одновременных пользователей | `1,5,10,20,30,50` |
| `--duration=15` | Секунд на каждую фазу | `15` |
| `--limit-download-bytes=65536` | Сколько байт скачивать при тесте загрузки (0 = без ограничения) | `65536` |
| `--timeout=15000` | Таймаут запроса, мс | `15000` |

## Примеры

```bash
# Быстрый тест с ограниченным скачиванием
node bench/load-test.cjs http://nas:3000 --phases=5,10,20 --duration=10

# Тест с полным скачиванием книг (серьёзная нагрузка на диск)
node bench/load-test.cjs http://nas:3000 --limit-download-bytes=0

# Мягкий тест — только 1 и 5 пользователей
node bench/load-test.cjs http://nas:3000 --phases=1,5 --duration=30
```

## Как запускать на NAS Synology

1. Установите Node.js через **Package Center** (или Docker).
2. Скопируйте файл `bench/load-test.cjs` на **другой** компьютер в той же сети.
3. Запустите с указанием IP вашего NAS:
   ```bash
   node bench/load-test.cjs http://192.168.1.100:3000
   ```

> **Важно:** запускайте тест с другой машины, а не с самого NAS — иначе нагрузка на CPU/диск будет некорректной.

## Интерпретация результатов

Скрипт выводит таблицу с фазами:

```
Пользователей │ RPS     │ Ошибок │ p90 latency │ Статус
──────────────┼─────────┼────────┼─────────────┼────────────────────
1             │ 12.5    │ 0      │ 120 ms      │ OK
5             │ 45.2    │ 0      │ 340 ms      │ OK
10            │ 78.1    │ 0      │ 890 ms      │ НАГРУЖЕН
20            │ 65.3    │ 12     │ 5200 ms     │ ДЕГРАДАЦИЯ
```

- **OK** — latency < 1 с, ошибок нет.
- **НАГРУЖЕН** — latency 1–3 с, но работает.
- **ДЕГРАДАЦИЯ** — ошибки или latency > 5 с.

Рекомендуемый лимит — последняя фаза без ошибок и с p90 < 3 с.
